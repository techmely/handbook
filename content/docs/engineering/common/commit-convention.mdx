## Git Commit Message Convention

We follow this: https://www.conventionalcommits.org/en/v1.0.0/

Read this carefully your self, and this some example for this:

Change/Add/Remove files on `apps/web` folder
   
   ```bash
   feat(web): add xyz to do something useful for end-user
   build(wasm): add Dockerfile more slightly for wasm
   chore(web): upgrade deps to latest 
   ci: Remove unnecessary script on build action
   docs(web): add docs in complicated function in xyz modules
   style(web): make the header more elegant to the end-user 
   ref(api): Move code to es6
   perf(web): use web-worker to reduce the times script execution on main thread
   test(api): add mock unit test for main function... 
   ```
